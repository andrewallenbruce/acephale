1 Advanced Operations

Create a new function call my_mean() that will take an argument, x, and then returns the results of enquo(x)

my_mean <- function(x) enquo(x)

Test the new function. It should return the same variable name, but inside a quosure. Use order_total as its argument’s value to test

my_mean(order_total)

Remove the last line that has x, add the contents of the function with the initial dplyr code from step Then replace order_total with !! x

my_mean <- function(x) {
  x <- enquo(x)
  orders |>
    summarise(mean = mean(!!x, na.rm = TRUE))
}

In the summarise() step, replace the name mean, with !! as_label(x), also replace the = sign, with :=

my_mean <- function(x){
  x <- enquo(x)
  
    summarise(
      !!as_label(x) := mean(!!x, na.rm = TRUE))
}

Run the function again, the name of the column should match the argument value

my_mean(order_total)

Test the function by passing a formula, such as order_total / order_qty

mean(order_total / order_qty)

Make the function generic, add a new argument called: .data. Inisde the function, replace orders with .data

my_mean <- function(.data, x) {
  x <- enquo(x)
  .data %>%
    summarise(!!as_label(x) := mean(!!x, na.rm = TRUE))
}
  1. The function now behaves more like a dplyr verb. Start with orders and then pipe into the function ::: {.cell layout-align=“center”}
orders %>%
  my_mean(order_total)

:::

  1. Clean up the code by removing the pipe that inside the function ::: {.cell layout-align=“center”}
my_mean <- function(.data, x) {
  x <- enquo(x)
  summarise(.data, !!as_label(x) := mean(!!x, na.rm = TRUE))
}

:::

  1. Confirm that there is no change in the behavior of the function ::: {.cell layout-align=“center”}

:::

  1. Add a show_query() step to preview the resulting SQL statement ::: {.cell layout-align=“center”}

:::

library(purrr)

  1. Use … as the second argument of a function called de_select(). Inside the function use enquos() to parse it ::: {.cell layout-align=“center”}
de_select <- function(.data, ...) {
  vars <- enquos(...)
  vars
}

:::

  1. Test the function using orders ::: {.cell layout-align=“center”}
orders %>%
  de_select(order_id, date)

:::

  1. Add a step to the function that iterates through each quosure and prefixes a minus sign to tell select() to drop that specific field. Use map() for the iteration, and quo() to create the prefixed expression. ::: {.cell layout-align=“center”}
de_select <- function(.data, ...) {
  vars <- enquos(...)
  vars <- map(vars, ~ quo(-!!.x))
  vars
}

:::

  1. Run the same test to view the new results ::: {.cell layout-align=“center”}
orders %>%
  de_select(order_id, date)

:::

  1. Add the select() step. Use !!! to parse the vars variable inside select() ::: {.cell layout-align=“center”}
de_select <- function(.data, ...) {
  vars <- enquos(...)
  vars <- map(vars, ~ quo(-!!.x))
  select(.data, !!!vars)
}

:::

1.2 Multiple queries

Suggested approach to avoid passing multiple, and similar, queries to the database

  1. Create a simple dplyr piped operation that returns the mean of order_total for the months of January, February and March as a group ::: {.cell layout-align=“center”}
orders %>%
  filter(date_month %in% c(1,2,3)) %>%
  summarise(mean = mean(order_total, na.rm = TRUE))

:::

  1. Assign the first operation to a variable called a, and create copy of the operation but changing the selected months to January, March and April. Assign the second one to a variable called b. ::: {.cell layout-align=“center”}
a <- orders %>%
  filter(date_month %in% c(1,2,3)) %>%
  summarise(mean = mean(order_total, na.rm = TRUE)) 

b <- orders %>%
  filter(date_month %in% c(1,3,4)) %>%
  summarise(mean = mean(order_total, na.rm = TRUE))

:::

  1. Use union() to pass a and b at the same time to the database ::: {.cell layout-align=“center”}
union(a, b)

:::

  1. Pipe the previous instruction to show_query() to confirm that the resulting query is a single one ::: {.cell layout-align=“center”}
union(a, b) %>%
  show_query()

:::

  1. Assign to a new variable called months an overlapping set of months ::: {.cell layout-align=“center”}
months <- list(
  c(1,2,3),
  c(1,3,4),
  c(2,4,6)
)

:::

  1. Use map() to cycle through each set of overlapping months. Notice that it returns three separate results, meaning that it went to the database three times ::: {.cell layout-align=“center”}
months %>%
  map(
    ~ orders %>%
        filter(date_month %in% .x) %>%
        summarise(mean = mean(order_total, na.rm = TRUE)) 
  )

:::

  1. Add a reduce() operation and use union() command to create a single query ::: {.cell layout-align=“center”}
months %>%
  map(
    ~ orders %>%
        filter(date_month %in% .x) %>%
        summarise(mean = mean(order_total, na.rm = TRUE)) 
  ) %>%
  reduce(function(x, y) union(x, y))

:::

  1. Use show_query() to see the resulting single query sent to the database ::: {.cell layout-align=“center”}
months %>%
  map(
    ~ orders %>%
        filter(date_month %in% .x) %>%
        summarise(mean = mean(order_total, na.rm = TRUE)) 
  ) %>%
  reduce(function(x, y) union(x, y)) %>%
  show_query()

:::

map2(


## Multiple queries
*Suggested approach to avoid passing multiple, and similar, queries to the database*

1. Create a simple `dplyr` piped operation that returns the mean of *order_total* for the months of January, February and March as a group
```{r}
orders %>%
  filter(date_month %in% c(1,2,3)) %>%
  summarise(mean = mean(order_total, na.rm = TRUE)) 
```

1. Assign the first operation to a variable called *a*, and create copy of the operation but changing the selected months to January, March and April.  Assign the second one to a variable called *b*.
```{r}
a <- orders %>%
  filter(date_month %in% c(1,2,3)) %>%
  summarise(mean = mean(order_total, na.rm = TRUE)) 

b <- orders %>%
  filter(date_month %in% c(1,3,4)) %>%
  summarise(mean = mean(order_total, na.rm = TRUE)) 
```

1. Use *union()* to pass *a* and *b* at the same time to the database
```{r}
union(a, b)
```

1. Pipe the previous instruction to `show_query()` to confirm that the resulting query is a single one
```{r}

  show_query()
```


1. Assign to a new variable called *months* an overlapping set of months
```{r}
months <- list(
  c(1,2,3),
  c(1,3,4),
  c(2,4,6)

```

```{r}
months %>%
  map(
    ~ orders %>%
        filter(date_month %in% .x) %>%
```
